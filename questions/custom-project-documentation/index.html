定制化项目如何开展，文档如何归档？

这个问题适合反向了解团队成熟度，也能展示自己对交付复用和文档标准化的关注。

定制化项目文档归档原型需求规格说明书

推荐回答

定制化项目我会先关注需求入口、业务口径、交付边界和文档归档。前期要确认客户需求是否能复用现有产品能力，不能复用的部分再拆成定制化范围；过程中用原型、需求规格说明书和会议纪要固化共识；交付后把需求、方案、原型、接口和验收材料按项目归档，方便后续维护和复用。

二面中的表现

你主动询问了原型工具、文档是否有标准化要求、文档是否按格式归档、定制化项目如何安排、整体项目如何开展，并给产品展示了自己画的原型和需求规格说明书。这是加分点，因为它证明你关注真实协作方式。

延伸准备

后续可以准备一套自己的文档样例说明：需求背景、业务流程、页面原型、字段说明、接口依赖、验收标准、变更记录。
